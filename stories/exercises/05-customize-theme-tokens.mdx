# 05. Customize with theme tokens

With content on the landing page, we're going to make sweeping changes using USWDS Design Tokens and theme settings.

You can view all available theme settings by visiting the [USWDS Settings Table](https://designsystem.digital.gov/documentation/settings/#general-settings-2).

USWDS theme color tokens are designed to be useful and effective for any project size. Let's start by updating some of our [theme color tokens](https://designsystem.digital.gov/documentation/settings/#color-settings-2).

1. Select either color palette (Earth tones or Oceanic) and copy the SCSS theme settings variables.

1. Visit `uswds-theme.scss`.
1. Open `uswds-theme.scss`. Uncomment the `@use 'uswds-core with (...)` section and paste the theme settings from the chosen palette between the parenthesis.
    ```scss
    // uswds-theme.scss
    @use "uswds-core" with (
      Theme color settings go here
    );
    ```
1. Save and allow storybook to rebuild. Visit the landing page and view your new theme!
1. There are many diverse theme settings available. For example, lets update the heavy font weight and increase the border radius tokens.
    ```scss
    $theme-font-weight-bold: 900,

    $theme-border-radius-sm: 1,
    $theme-border-radius-md: 2,
    $theme-border-radius-lg: 3,
    ```
1. Refresh the page and we'll see the hero heading with the increased font weights, and the corners of the components are more round.

    If you take a look at the search input within the header, you might have noticed the imbalanced border radius. Since the button border radius is not used on the input, we'll have to update manually.

    To showcase another great USWDS tool, lets add a utility class to the `input` element.

1. Visit the [border utility](https://designsystem.digital.gov/utilities/border/) page
1. In the "examples and usage" section, find the find the class that utilizes the **`md` token for the left border radius**.
1. Open `/stories/components/header/header.html.twig`.
1. Add the class to the `usa-input` input element tag.
1. View the header and you'll see the border radius now matches the button! Since we updated the `md` border radius value, the change will be reflected _everywhere_ the `md` border radius is used!
